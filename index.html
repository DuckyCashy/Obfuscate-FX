      Heads up — this runs lightweight, regex-based transforms rather than a full parser for each language.
      Light and Medium levels are safe for almost any input. Heavy renames identifiers with lookalike
      characters and can occasionally catch a name it shouldn't (e.g. inside an unusual string). C/C++ string
      encoding uses compound literals, which need a C99-or-newer compiler. Lua/Luau string decoding uses
      bit32.bxor, so it needs Lua 5.2+ or Luau — plain Lua 5.1 without the bit32 library should
      stick to Light. Always diff or test-run obfuscated output before shipping it.
    

    

      On "undeobfuscatable" — no obfuscator, this one included, can make functional code truly
      unrecoverable. If code has to run, its logic and every string it ever produces can be extracted just by
      running it. Medium/Heavy now XOR-encode string literals with a random per-run key before packing them —
      that defeats simple find-and-decode scripts and casual reading, which covers most real-world "script
      stealing." A dedicated reverse engineer with time on their hands can still get through it. Treat this as
      raising the cost of copying your work, not as encryption.
